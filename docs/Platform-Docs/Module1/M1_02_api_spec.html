API Spec

Request bodies, response schemas, validation rules, and error codes for all Phase 1 routes.

Standard response envelope — all routes use this format:
Success: { "success": true, "data": {...}, "meta": { "page": 1, "total": 50 } }
Error: { "success": false, "error": { "code": "INVALID_CREDENTIALS", "message": "Invalid email or password." } }
Paginated lists include: meta: { page, limit, total, totalPages }

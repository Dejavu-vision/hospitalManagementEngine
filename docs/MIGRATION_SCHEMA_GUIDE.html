CuraMatrix HSM DB Schema

Data Migration Schema Guide

Consolidated 50-table topological database guide & sequence templates optimized for safe data imports.

💡 Tip: To avoid foreign key constraint errors during bulk imports, insert data in the topological order shown below, or execute SET FOREIGN_KEY_CHECKS = 0; prior to loading.

1. Migration Sequence (Topological Sort Order)

Due to complex foreign key relationships, insert records in the exact phase sequence below:

  1. Phase 1 (Global Lookups): tenants, roles, ui_pages, medicines, blocked_tokens
  2. Phase 2 (Accounts & Staff): users, user_roles, user_pages, role_pages, user_access_audit, departments, doctors, doctor_availability, doctor_status_log, receptionists
  3. Phase 3 (Hospital Setup): wards, rooms, beds, lab_services, hospital_services, service_pricing_tiers, payer_master, walk_in_token_sequence, employee_id_sequence, ipd_admission_sequence
  4. Phase 4 (Patients): patients, patient_registrations, patient_financial_account, insurance_policies
  5. Phase 5 (OPD & General): appointments, appointment_status_log, ipd_vital_signs, financial_blocks, diagnoses, prescriptions, medicine_inventory
  6. Phase 6 (In-Patient Stays): ipd_admissions, ipd_daily_progress_notes, bed_allocations, pre_auth_requests, pre_auth_coverage_items
  7. Phase 7 (Lab Tests): lab_prescriptions, lab_tests, lab_results, lab_documents, lab_test_status_log
  8. Phase 8 (Invoices & Receipts): billings, billing_items, bill_insurance_splits, payments, payment_plans, bill_allocations

Every API request passes through these layers in order
Layer 1
Rate limiter
rateLimit({ max: 100, window: '15m' }) — Redis backed
General: 100 req / 15 min per IPLogin endpoint: 5 attempts / 15 min per IPExceeded → 429 Too Many RequestsCounter stored in Redis with TTL
↓
Layer 2
JWT verifier
verifyJWT(token, JWT_SECRET) → decode payload
Extract token from Authorization: Bearer or httpOnly cookieVerify signature with JWT_SECRETCheck exp claim (not expired)Invalid / expired → 401 UnauthorizedDecoded: { userId, role, hospitalId, jti }
↓
Layer 3
Blacklist check
Redis.get(`blacklist:${jti}`) → reject if found
Lookup jti in Redis blacklistIf key exists → token was logged out or revokedFound → 401 "Token revoked"Not found → continueThis is why logout is instant even with JWT
↓
Layer 4
Permission check
checkPermission(role, route, method) → allow / deny
Look up permissions[role][route][method]Permission matrix defined as config objectNot permitted → 403 ForbiddenPermitted → continueNever 401 here — user is authenticated, just not authorized
↓
Layer 5
Branch scope injector
req.branchId = user.hospitalId → all DB queries scoped
Inject hospitalId from JWT into request contextAll DB queries automatically add WHERE hospital_id = req.branchIdSuper admin: no scope restriction (all branches)Admin/Staff: always scoped to own branch — cannot escape
↓
Layer 6
Audit logger
logAction(userId, action, resource, meta) — async write
Runs after controller completes (post-middleware)Writes: user_id, action, resource_id, ip, timestampFor mutations: writes old_value and new_valueAsync — does not slow down responseAppend-only — audit_logs table has no UPDATE/DELETE grants
↓
Success
200 / 201 — controller executes
Returns { success: true, data: {...}, meta: {...} }
Failure
401 / 403 / 429
Returns { success: false, error: "..." }

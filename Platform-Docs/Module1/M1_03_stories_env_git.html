Authentication
AUTH-01 Super admin logs in and lands on SA dashboard Super admin ▼
As a super admin, I want to log in with my email and password so I can access the platform management dashboard.
Acceptance criteria
✓Given valid credentials, login succeeds and I am redirected to /superadmin/dashboard
✓My name and role "Super admin" are visible in the dashboard header
✓A JWT access token is set (not visible in localStorage — httpOnly cookie)
✗Given wrong password, I see "Invalid email or password" — the error does not specify which field is wrong
✗After 5 failed attempts, I see "Too many attempts. Try again in 15 minutes" and cannot try again
✗If my account is deactivated, I see "Your account has been deactivated"
✗An admin visiting /superadmin/dashboard is redirected to /admin/dashboard
AUTH-02 Admin logs in and lands on admin dashboard Admin ▼
As an admin, I want to log in and be taken to my branch dashboard, not the super admin panel.
✓Valid login redirects to /admin/dashboard
✓Dashboard shows my hospital name and branch code
✓I can only see my branch's data — no other hospitals visible
✗Navigating to /superadmin/dashboard shows "Access denied"
✗Calling GET /api/hospitals (SA-only) returns 403
AUTH-03 User resets forgotten password All roles ▼
As any user, I want to reset my password via email if I forget it.
✓I enter my email and always see "If that email exists, a link was sent" — never a confirmation
✓I receive an email with a reset link (expires in 1 hour)
✓I set a new password meeting complexity rules and can log in immediately
✓All my other active sessions are logged out after reset
✗Clicking an expired link shows "This link has expired. Request a new one."
✗Clicking an already-used link shows "This link has already been used."
✗Entering a non-existent email shows the same success message — no user enumeration
AUTH-04 User logs out and session is invalidated All roles ▼
As any user, I want logging out to immediately end my session so no one can use my browser.
✓Clicking logout redirects to /login immediately
✗After logout, using the old JWT token returns 401 "Token revoked"
✗Pressing browser back after logout does not restore the authenticated view
Hospital onboarding — super admin
HOSP-01 Super admin creates a new hospital Super admin ▼
As a super admin, I want to create a new hospital so it can be onboarded to the platform.
✓I fill name, address, city, branch code, plan and submit
✓Hospital appears in list with status "Pending"
✓A settings row is auto-created with all checklist items = false
✓Action is logged in audit trail
✗Using a duplicate branch code shows "Branch code already in use"
✗Admin cannot access the create hospital form — they see 403
HOSP-02 Super admin assigns an admin to a hospital Super admin ▼
As a super admin, I want to assign an admin to a hospital so they can configure and manage it.
✓I enter an email address and select role "Admin" — invite is sent
✓Admin receives invite email with a link valid for 48 hours
✓Checklist item "admin_assigned" turns true
✗Using an already-registered email shows "User already exists"
✗Admin cannot invite another admin — form does not show "Admin" as a role option
HOSP-03 Super admin activates a hospital Super admin ▼
As a super admin, I want to activate a hospital so patients can start booking appointments.
✓When all 5 checklist items are complete, "Activate" button is enabled
✓Clicking activate changes status to "Active" and logs the action
✗With incomplete checklist, "Activate" button is disabled with tooltip "Complete onboarding first"
✗API call to activate with incomplete checklist returns 400 listing missing items
Admin — staff management
STAFF-01 Admin invites a doctor to the branch Admin ▼
As an admin, I want to invite a doctor so they can access the system and see patients.
✓I enter email + select "Doctor" from roles — invite sent
✓Doctor appears in staff list with status "Invited"
✓Doctor receives invite email valid for 48 hours
✓Checklist item "doctor_added" turns true
✗Role dropdown does not show "Admin" or "Super admin" options
✗Inviting someone from Hospital B's data is impossible — admin is scoped to own branch
STAFF-02 New staff accepts invite and sets password New staff member ▼
As a newly invited staff member, I want to click the invite link and set my password to activate my account.
✓Clicking link shows my role and hospital name pre-filled
✓I set name + password and am auto-logged in and redirected to my dashboard
✓My status in staff list changes from "Invited" to "Active"
✗Clicking an expired link (48h) shows "Invite expired. Contact your admin."
✗Clicking an already-used link shows "This invite has already been used."
✗Weak password (no uppercase/number/special char) shows inline validation error
STAFF-03 Admin deactivates a staff member Admin ▼
As an admin, I want to deactivate a staff member so they immediately lose access.
✓Staff appears as "Deactivated" in list immediately
✓Deactivated user's next API call returns 401 immediately (sessions killed)
✓Action is logged in audit trail with who did it and when
✗Admin cannot deactivate themselves — button is disabled for own account
✗Admin cannot deactivate staff from another branch
Edge cases & security
SEC-01 Cross-branch access attempt is blocked Admin ▼
As a security requirement, no admin should be able to read or write data belonging to another hospital.
✗GET /api/hospitals/[other-hospital-id]/staff returns 403
✗PATCH /api/settings/hospital/[other-hospital-id] returns 403
✗GET /api/audit/logs/hospital/[other-hospital-id] returns 403
✗All cross-branch attempts are logged in audit_logs with action UNAUTHORIZED_ACCESS_ATTEMPT
SEC-02 Rate limiting on login endpoint All ▼
As a security requirement, brute force attacks on the login endpoint must be blocked.
✓5 failed logins from same IP within 15 minutes triggers rate limit
✗6th attempt returns 429 with message "Too many attempts. Try again in X minutes."
✗Response includes Retry-After header with seconds remaining
✓Counter resets after 15 minutes

Branch strategy
BranchPurposeRules
mainProduction-ready code onlyNo direct push. Merge from staging only. Protected.
stagingQA and testing environmentMerge from dev after QA sign-off. Auto-deploys to staging server.
devIntegration branchFeature branches merge here. Runs CI on every push.
feature/TASK-ID-nameIndividual feature worke.g. feature/API-02-login-endpoint. Branches off dev. PR back to dev.
fix/TASK-ID-nameBug fixese.g. fix/AUTH-01-token-expiry. Branches off dev (or main for hotfixes).
Commit message format
feat(auth): add POST /login endpoint with rate limiting feat(hospitals): create hospital with checklist auto-init fix(auth): correct token blacklist TTL calculation test(middleware): add unit tests for checkPermission chore(db): add migration 006 for audit_logs table docs(api): update invite endpoint response schema
Format: type(scope): short description — types: feat | fix | test | chore | docs | refactor
PR rules
Before opening a PR
  • All tests pass locally
  • No console.log statements left in code
  • No .env file committed (check with git status)
  • PR title matches commit format: feat(auth): ...
  • PR description links the task ID: "Closes DB-01"
Review requirements
  • Minimum 1 reviewer approval before merge
  • CI pipeline must be green (lint + tests)
  • No direct push to dev, staging, or main
  • Reviewer checks: does the code match the API spec and task acceptance criteria?
.gitignore must include
  • .env (all variants — .env.local, .env.production)
  • node_modules/
  • *.log files
  • dist/ and .next/ build folders
  • *.pem and *.key certificate files
